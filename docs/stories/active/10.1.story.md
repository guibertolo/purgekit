# Story 10.1: Snapshot de Estado do Sistema e Mecanismo de Restore

## Metadata
- **Epic:** 10 - Gaming Mode
- **Status:** Draft
- **Priority:** P0
- **Story Points:** 5
- **Depends On:** 1.3
- **FR References:** FR-063, FR-068

## User Story
Como usuario gamer, eu quero que o app salve o estado atual do sistema ao ativar o Gaming Mode e restaure tudo ao desativar, para que eu nao precise reconfigurar manualmente servicos e configuracoes apos cada sessao de jogo.

## Description
Implementar o mecanismo de snapshot/restore que e a fundacao do Gaming Mode. Antes de qualquer modificacao (desativar servicos, alterar Registry), o app salva o estado atual em `state/gaming_snapshot.rs`. Ao desativar, restaura tudo a partir do snapshot. Se o app crashar com Gaming Mode ativo, na proxima abertura detecta o snapshot e oferece restaurar.

## Acceptance Criteria
- [ ] AC-1: `GamingSnapshot` struct serializa: lista de servicos com estado (Running/Stopped/Disabled), configuracoes de Registry modificadas, timestamp de ativacao
- [ ] AC-2: Snapshot e salvo em `%APPDATA%\WindowsOptimizer\gaming-snapshot.json` ao ativar Gaming Mode
- [ ] AC-3: `state/gaming_snapshot.rs` expoe: `save_snapshot()`, `load_snapshot()`, `clear_snapshot()`, `snapshot_exists()`
- [ ] AC-4: `AppState` tem campo `gaming_mode_active: Arc<AtomicBool>` para indicar Gaming Mode ativo
- [ ] AC-5: Tauri command `get_gaming_mode_status` retorna `{ active: bool, snapshot_exists: bool, activated_at: Option<i64> }`
- [ ] AC-6: Ao iniciar o app, verificar se `gaming-snapshot.json` existe — se sim, exibir aviso no frontend: "Gaming Mode estava ativo. Deseja restaurar o sistema?"
- [ ] AC-7: Tauri command `restore_from_snapshot` le o snapshot, restaura servicos e Registry ao estado salvo, deleta o snapshot, seta `gaming_mode_active = false`
- [ ] AC-8: Restore de servicos requer elevacao admin
- [ ] AC-9: Cada item restaurado gera entrada de log
- [ ] AC-10: Se snapshot estiver corrompido/invalido, logar erro e limpar snapshot — nao crashar

## Technical Notes

### GamingSnapshot struct (state/gaming_snapshot.rs)
```rust
#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub struct GamingSnapshot {
    pub activated_at: i64,          // unix timestamp
    pub services: Vec<ServiceSnapshot>,
    pub registry_entries: Vec<RegistrySnapshot>,
}

#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub struct ServiceSnapshot {
    pub name: String,
    pub original_state: String,     // "Running", "Stopped"
    pub original_startup: String,   // "Auto", "Manual", "Disabled"
}

#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub struct RegistrySnapshot {
    pub hive: String,               // "HKCU", "HKLM"
    pub path: String,
    pub value_name: String,
    pub original_value: String,     // JSON-encoded do valor original
    pub original_type: String,      // "DWORD", "SZ", etc.
}
```

### Persistencia do snapshot
```rust
impl GamingSnapshot {
    pub fn save(&self, data_dir: &Path) -> Result<(), AppError> {
        let path = data_dir.join("gaming-snapshot.json");
        let json = serde_json::to_string_pretty(self)?;
        std::fs::write(&path, json)?;
        Ok(())
    }

    pub fn load(data_dir: &Path) -> Result<Option<Self>, AppError> {
        let path = data_dir.join("gaming-snapshot.json");
        if !path.exists() { return Ok(None); }
        let content = std::fs::read_to_string(&path)?;
        match serde_json::from_str(&content) {
            Ok(snapshot) => Ok(Some(snapshot)),
            Err(e) => {
                tracing::error!("Gaming snapshot corrupted: {}. Clearing.", e);
                let _ = std::fs::remove_file(&path);
                Ok(None)
            }
        }
    }

    pub fn clear(data_dir: &Path) -> Result<(), AppError> {
        let path = data_dir.join("gaming-snapshot.json");
        if path.exists() { std::fs::remove_file(&path)?; }
        Ok(())
    }
}
```

### Verificacao no startup
Em `main.rs`, apos init do Tauri e antes de mostrar a janela, verificar `GamingSnapshot::load()`. Se existir, setar flag em `AppState` para que o frontend exiba o aviso de restore.

O restore itera pelos `ServiceSnapshot` e chama `set_service_startup_type(name, original_startup)` e, se necessario, `start_service(name)` ou `stop_service(name)`.
