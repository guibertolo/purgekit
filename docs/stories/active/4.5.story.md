# Story 4.5: Dashboard Unificado com Graficos Real-Time

## Metadata
- **Epic:** 4 - System Monitoring Dashboard
- **Status:** Done
- **Priority:** P0
- **Story Points:** 5
- **Depends On:** 4.1, 4.2, 4.3, 4.4
- **FR References:** FR-040, FR-041, FR-042, FR-045, FR-046, FR-048, NFR-003, NFR-012

## User Story
Como usuario do Windows Optimizer, eu quero um dashboard unificado com graficos de CPU, RAM e GPU atualizando em tempo real, para que eu tenha uma visao geral da performance do sistema em uma unica tela.

## Description
Implementar a pagina `Dashboard.tsx` como tela home do app, integrando todos os monitores das Stories 4.1-4.4 em um layout coeso. Graficos de linha Recharts para CPU e GPU, barras para RAM, cards de temperatura, e informacoes de disco (espaco livre por drive via sysinfo). Layout responsivo e dark theme com estetica gamer discreta (NFR-003). Honest UI (NFR-012) — dados reais, sem dramatismo.

### Deteccao de discos para FR-048
```rust
// Em modules/system_monitor.rs
use sysinfo::DiskExt;

pub fn build_disk_info(sys: &System) -> Vec<DiskInfo> {
    sys.disks().iter().map(|d| DiskInfo {
        name: d.name().to_string_lossy().into_owned(),
        mount_point: d.mount_point().to_string_lossy().into_owned(),
        total_bytes: d.total_space(),
        used_bytes: d.total_space() - d.available_space(),
        free_bytes: d.available_space(),
        usage_percent: ((d.total_space() - d.available_space()) as f32 / d.total_space() as f32) * 100.0,
    }).collect()
}
```

### Recharts — grafico de CPU
```typescript
import { LineChart, Line, XAxis, YAxis, Tooltip, ResponsiveContainer } from 'recharts';

// data = cpuHistory: Array<{ time: number; value: number }>
<ResponsiveContainer width="100%" height={120}>
  <LineChart data={cpuHistory}>
    <Line type="monotone" dataKey="value" stroke="var(--accent-blue)" dot={false} isAnimationActive={false} />
    <YAxis domain={[0, 100]} hide />
    <Tooltip formatter={(v) => `${v}%`} />
  </LineChart>
</ResponsiveContainer>
```

`isAnimationActive={false}` e crucial para evitar flickering em atualizacoes frequentes.

### Skeleton loading
Usar `MetricCard` com estado `isLoading: true` enquanto `metricsStore.latest === null`.

### Honest UI (NFR-012)
- Nenhum "seu PC esta lento!" mesmo com CPU a 90%
- Sem mascotes tristes ou animacoes de susto
- Labels factuais: "CPU: 90%" nao "CPU critica!"
- AlertBanner de temperatura: "CPU: 92°C — acima do limite de 85°C" — factual e direto
