# Story 9.1: Deteccao de Ferramentas de Desenvolvimento

## Metadata
- **Epic:** 9 - Developer Mode
- **Status:** Draft
- **Priority:** P0
- **Story Points:** 3
- **Depends On:** 1.3
- **FR References:** FR-053

## User Story
Como usuario desenvolvedor, eu quero que o app detecte automaticamente quais ferramentas de desenvolvimento estao instaladas na minha maquina, para que apenas as opcoes de limpeza relevantes para meu ambiente sejam exibidas.

## Description
Implementar `modules/dev_cleaner.rs` com deteccao automatica de ferramentas de desenvolvimento instaladas. A deteccao e feita via verificacao de executaveis no PATH e paths conhecidos — nao via Registry. Retorna lista de ferramentas detectadas com versao quando disponivel.

Ferramentas a detectar (FR-053): Node.js (npm/pnpm/yarn), Python (pip), Rust (cargo), Go, Java (Maven/Gradle), Docker, WSL2, VS Code, JetBrains IDEs.

## Acceptance Criteria
- [ ] AC-1: Tauri command `detect_dev_tools` retorna `Vec<DevToolInfo>` com cada ferramenta detectada
- [ ] AC-2: `DevToolInfo` inclui: `id: String`, `name: String`, `detected: bool`, `version: Option<String>`, `cache_paths: Vec<String>`
- [ ] AC-3: Node.js detectado se `node.exe` encontrado no PATH — versao via `node --version`
- [ ] AC-4: npm detectado se `npm.cmd` no PATH; pnpm se `%LOCALAPPDATA%\pnpm\` existe; yarn se `%LOCALAPPDATA%\Yarn\` existe

- [ ] AC-6: Rust/cargo detectado se `%USERPROFILE%\.cargo\bin\cargo.exe` existe

- [ ] AC-9: WSL2 detectado se `wsl.exe` existe em `%WINDIR%\System32\`
- [ ] AC-10: Deteccao completa executa em menos de 3 segundos

## Technical Notes

### DevToolInfo struct
```rust
#[derive(Debug, Clone, serde::Serialize)]
pub struct DevToolInfo {
    pub id: String,              // "nodejs", "npm", "pnpm", "yarn", "python", etc.
    pub name: String,            // "Node.js", "npm", etc.
    pub detected: bool,
    pub version: Option<String>,
    pub cache_paths: Vec<String>, // paths de cache associados a esta ferramenta
}
```

### Estrategia de deteccao
1. **PATH lookup**: usar `std::process::Command::new("where").arg(executable)` ou verificar existencia direta
2. **Path direto**: verificar existencia de diretorio/executavel em path conhecido
3. **Versao**: executar `executable --version` com timeout de 1s

```rust
pub fn detect_tool(executable: &str) -> Option<String> {
    // Tenta executar `executable --version` com timeout
    use std::process::Command;
    let output = Command::new(executable)
        .arg("--version")
        .output()
        .ok()?;
    if output.status.success() {
        Some(String::from_utf8_lossy(&output.stdout).trim().to_string())
    } else {
        None
    }
}

pub fn path_exists(path_template: &str) -> bool {
    let expanded = expand_env_path(path_template);
    std::path::Path::new(&expanded).exists()
}
```

### Cache paths por ferramenta
Cada `DevToolInfo.cache_paths` lista os paths de cache que serao limpos nas stories subsequentes:
- npm: `["%APPDATA%\npm-cache"]`
- pnpm: `["%LOCALAPPDATA%\pnpm-store", "%LOCALAPPDATA%\pnpm\store"]`
- yarn: `["%LOCALAPPDATA%\Yarn\Cache"]`
- pip: `["%LOCALAPPDATA%\pip\Cache"]`
- cargo: `["%USERPROFILE%\.cargo\registry\cache", "%USERPROFILE%\.cargo\registry\src"]`
- go: `["%USERPROFILE%\go\pkg\mod"]`
- maven: `["%USERPROFILE%\.m2\repository"]`
- gradle: `["%USERPROFILE%\.gradle\caches"]`

### Docker — deteccao extra
Para Docker, verificar nao apenas o executavel mas se o daemon esta respondendo:
```rust
fn is_docker_running() -> bool {
    Command::new("docker").arg("info").output()
        .map(|o| o.status.success())
        .unwrap_or(false)
}
```
