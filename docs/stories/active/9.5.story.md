# Story 9.5: UI do Developer Mode

## Metadata
- **Epic:** 9 - Developer Mode
- **Status:** Draft
- **Priority:** P0
- **Story Points:** 5
- **Depends On:** 1.2, 9.1, 9.2, 9.3, 9.4
- **FR References:** FR-062, NFR-004, NFR-012

## User Story
Como desenvolvedor, eu quero uma interface consolidada que mostre todas as oportunidades de limpeza do meu ambiente de desenvolvimento com estimativa total de espaco, para que eu selecione e execute as limpezas com um fluxo claro.

## Description
Implementar a pagina `DeveloperMode.tsx` completa integrando todas as funcionalidades das Stories 9.1-9.4. Interface com: secao de ferramentas detectadas, tabela consolidada de caches com espaco estimado, scan de node_modules com status, estimativa total, e fluxo de confirmacao + limpeza com progresso.

## Acceptance Criteria
- [ ] AC-1: Topo da pagina exibe lista de ferramentas detectadas com badges (Node.js 20.x, cargo 1.x, Docker, etc.)
- [ ] AC-2: Botao "Scan Dev Caches" dispara todos os scans em paralelo e exibe progresso
- [ ] AC-3: Apos scan, exibe tabela consolidada (FR-062): [Categoria | Ferramenta | Path | Tamanho | Status | Checkbox]
- [ ] AC-4: Secao separada para `node_modules`: lista com status (Active/Inactive/Orphan), tamanho, ultima modificacao, checkbox
- [ ] AC-5: Footer exibe total geral de espaco selecionado para limpeza
- [ ] AC-6: Categorias de ferramentas nao detectadas aparecem em estado disabled com "Ferramenta nao instalada"
- [ ] AC-7: Secao WSL2 aparece apenas se WSL2 detectado — exibe distros com tamanho do vhdx e botao "Compactar"
- [ ] AC-8: "Compactar WSL2" abre ConfirmDialog com aviso de que WSL2 sera desligado durante o processo
- [ ] AC-9: "Select All" e "Deselect All" disponivel (FR-062)
- [ ] AC-10: Botao "Limpar Dev Caches" executa limpeza das categorias selecionadas com ProgressOverlay

### Tipos TypeScript (types/cleaner.ts — expandir)
```typescript
export interface NodeModulesEntry {
  path: string;
  parentPath: string;
  sizeBytes: number;
  lastModified: number;
  hasPackageJson: boolean;
  status: 'Active' | { Inactive: { daysOld: number } } | 'Orphan';
  selected: boolean;
}

export interface DevCacheCategory {
  id: string;
  tool: string;
  name: string;
  paths: string[];
  sizeBytes: number;
  available: boolean;
}

export interface Wsl2DistroInfo {
  name: string;
  vhdxPath: string;
  vhdxSizeBytes: number;
  status: string;
}
```

### Scans em paralelo
Ao clicar "Scan Dev Caches", disparar todos os scans em paralelo:
```typescript
const [jsCaches, languageCaches, nodeModules, wsl2] = await Promise.allSettled([
  invoke<DevCacheCategory[]>('scan_js_caches'),
  invoke<DevCacheCategory[]>('scan_language_caches'),
  invoke<NodeModulesEntry[]>('scan_node_modules', { scanDirs, inactiveThresholdDays: 90 }),
  invoke<Wsl2DistroInfo[]>('detect_wsl2_distributions'),
]);
```

### Feedback de scan de node_modules (lento)
O scan de node_modules pode ser lento (muitos diretorios). Usar `listen("dev-scan-progress", ...)` para atualizar um contador em tempo real: "Encontrados: X node_modules..."

### Honest UI — total de espaco
Mostrar o total REAL como calculado. Nao arredondar para cima ou inflar. "Total selecionado: 16.2 GB" — factual.
