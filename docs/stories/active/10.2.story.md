# Story 10.2: Desativacao Temporaria de Servicos para Gaming

## Metadata
- **Epic:** 10 - Gaming Mode
- **Status:** Draft
- **Priority:** P0
- **Story Points:** 5
- **Depends On:** 10.1
- **FR References:** FR-064, FR-068

## User Story
Como usuario gamer, eu quero que o Gaming Mode desative servicos desnecessarios temporariamente, para que eu libere recursos de CPU e RAM durante o jogo.

## Description
Implementar `modules/gaming_mode.rs` com a logica de desativacao de servicos para gaming. Antes de desativar, salva o estado atual no `GamingSnapshot` (Story 10.1). A lista de servicos e configuravel. Nao desativa servicos classificados como DANGEROUS. Requer elevacao admin. Ao desativar o Gaming Mode, restaura todos os servicos ao estado original.

## Acceptance Criteria
- [ ] AC-1: `modules/gaming_mode.rs` implementa `activate_gaming_services()` que desativa lista de servicos e salva estado no snapshot
- [ ] AC-2: Servicos desativados por default: DiagTrack, WSearch (Windows Search), SysMain, Connected User Experiences (DiagTrackSvc), DownloadManager (Downloaded Maps), Spooler (se impressora nao ativa)
- [ ] AC-3: Lista de servicos e configuravel via `AppConfig` — usuario pode adicionar ou remover
- [ ] AC-4: Nenhum servico classificado como DANGEROUS e desativado (FR-064)
- [ ] AC-5: Tauri command `activate_gaming_mode` requer elevacao admin — retorna `AppError::ElevationRequired` se sem admin
- [ ] AC-6: Antes de ativar, retorna `GamingPreview` com lista do que sera feito (para confirmacao no frontend)
- [ ] AC-7: Para cada servico desativado: registra `ServiceSnapshot` no `GamingSnapshot` com estado original (Running/Stopped, Auto/Manual/Disabled)
- [ ] AC-8: `restore_from_snapshot` (Story 10.1) restaura servicos ao estado original — se estava Running, reinicia; se estava Stopped, deixa stopped
- [ ] AC-9: Logging de cada servico desativado e restaurado via `tracing::info!`
