# Story 1.1: Scaffold Tauri 2.x Project

## Metadata
- **Epic:** 1 - Foundation & Tauri Project Setup
- **Status:** InProgress
- **Priority:** P0
- **Story Points:** 5
- **Depends On:** None
- **FR References:** CON-001, NFR-001, NFR-002, NFR-007, NFR-008

## User Story
Como desenvolvedor do Windows Optimizer, eu quero um projeto Tauri 2.x funcional com backend Rust e frontend React/TypeScript, para que toda implementacao subsequente tenha uma base solida e compilavel.

## Description
Criar o scaffold completo do projeto `windows-optimizer/` conforme a estrutura definida na arquitetura. O projeto deve compilar e rodar com uma tela canary minimal ("Windows Optimizer is running"), estabelecendo todos os modulos Rust, entrypoints React, configuracoes de build, e dependencias base.

A estrutura de diretorios deve seguir exatamente o definido na arquitetura:
- `src-tauri/` com modulos `commands/`, `modules/`, `platform/`, `config/`, `state/`, `error.rs`, `logging.rs`
- `src/` com `components/`, `pages/`, `hooks/`, `stores/`, `types/`, `lib/`, `styles/`
- Todos os arquivos Rust criados com stubs vazios (apenas `pub mod` declarations e structs placeholder)
- Todos os arquivos React criados com stubs (componentes com `export default function X() { return null }`)

## Acceptance Criteria
- [x] AC-1: `cargo build` executa sem erros com todos os modulos declarados em `lib.rs`

- [x] AC-3: A estrutura de diretorios corresponde exatamente ao definido na arquitetura (secao 2)

### AppError enum (error.rs)
Criar enum unificado com variantes iniciais:
```rust
#[derive(Debug, thiserror::Error, serde::Serialize)]
pub enum AppError {
    #[error("Elevation required for: {operation}")]
    ElevationRequired { operation: String },
    #[error("IO error: {0}")]
    Io(String),
    #[error("Not found: {0}")]
    NotFound(String),
}
```
