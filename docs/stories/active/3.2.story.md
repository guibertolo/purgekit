# Story 3.2: Scan e Limpeza de Shader Cache NVIDIA

## Metadata
- **Epic:** 3 - GPU Cache Cleaner
- **Status:** Draft
- **Priority:** P0
- **Story Points:** 3
- **Depends On:** 3.1
- **FR References:** FR-013, FR-016, FR-017

## User Story
Como usuario com GPU NVIDIA, eu quero escanear e limpar os shader caches da NVIDIA (DXCache, GLCache, NV_Cache), para que eu recupere espaco em disco ocupado por shaders compilados obsoletos.

## Description
Implementar scan e limpeza dos tres caches de shader NVIDIA no `modules/gpu_cleaner.rs`. Nao requer elevacao admin. Exibe espaco total antes da limpeza. Inclui aviso de que loading times de jogos podem aumentar temporariamente apos limpeza (FR-017).

## Acceptance Criteria
- [ ] AC-1: Scan de NVIDIA shader cache retorna tamanho de tres paths:
  - `%LOCALAPPDATA%\NVIDIA\DXCache\`
  - `%LOCALAPPDATA%\NVIDIA\GLCache\`
  - `%LOCALAPPDATA%\NVIDIA Corporation\NV_Cache\`
- [ ] AC-2: Cada path e reportado como sub-item com tamanho individual e tamanho total somado
- [ ] AC-3: Se NVML nao estiver disponivel, shader cache NVIDIA ainda e escaneado via filesystem (nao depende de NVML para scan)
- [ ] AC-4: Limpeza nao requer elevacao admin
- [ ] AC-5: `GpuCacheCleanResult` inclui `first_load_slower_warning: true` apos limpeza de shader cache
- [ ] AC-6: Scan retorna `requires_nvidia: true` — se nenhuma GPU NVIDIA detectada, item aparece como indisponivel

## Technical Notes

### Paths NVIDIA (FR-013)
```rust
const NVIDIA_SHADER_CACHE_PATHS: &[(&str, &str)] = &[
    ("dx_cache",  r"%LOCALAPPDATA%\NVIDIA\DXCache"),
    ("gl_cache",  r"%LOCALAPPDATA%\NVIDIA\GLCache"),
    ("nv_cache",  r"%LOCALAPPDATA%\NVIDIA Corporation\NV_Cache"),
];
```

### GpuCacheCategory struct
```rust
#[derive(Debug, Clone, serde::Serialize)]
pub struct GpuCacheCategory {
    pub id: String,
    pub name: String,
    pub vendor: String,    // "NVIDIA", "AMD", "DirectX"
    pub paths: Vec<GpuCachePath>,
    pub total_size_bytes: u64,
    pub available: bool,   // false se vendor nao detectado
    pub requires_elevation: bool, // sempre false para shader cache
}

#[derive(Debug, Clone, serde::Serialize)]
pub struct GpuCachePath {
    pub label: String,
    pub path: String,
    pub size_bytes: u64,
    pub exists: bool,
}
```

### scan_nvidia_shader_cache() em modules/gpu_cleaner.rs
```rust
pub fn scan_nvidia_shader_cache(
    has_nvidia: bool,
) -> GpuCacheCategory {
    let mut category = GpuCacheCategory {
        id: "nvidia_shader".into(),
        name: "NVIDIA Shader Cache".into(),
        vendor: "NVIDIA".into(),
        paths: vec![],
        total_size_bytes: 0,
        available: has_nvidia,
        requires_elevation: false,
    };

    if !has_nvidia {
        return category;
    }

    for (label, path_template) in NVIDIA_SHADER_CACHE_PATHS {
        let path = expand_env_path(path_template);
        let (size, _) = calc_dir_size(Path::new(&path));
        category.paths.push(GpuCachePath {
            label: label.to_string(),
            path: path.clone(),
            size_bytes: size,
            exists: Path::new(&path).exists(),
        });
        category.total_size_bytes += size;
    }

    category
}
```

### Nota: scan nao depende de NVML
O scan e puramente filesystem — verifica existencia dos paths e calcula tamanho. NVML e necessario apenas para monitoramento em tempo real (Epic 4). Para o scan de shader cache, usar `has_nvidia` booleano derivado da deteccao de GPU (Story 3.1).
