# Story 1.3: Logging Persistente e Config Store

## Metadata
- **Epic:** 1 - Foundation & Tauri Project Setup
- **Status:** Done
- **Priority:** P0
- **Story Points:** 3
- **Depends On:** 1.1
- **FR References:** NFR-006, NFR-005

## User Story
Como usuario do Windows Optimizer, eu quero que todas as operacoes sejam logadas em arquivo persistente e que minhas preferencias sejam salvas entre sessoes, para que eu tenha rastreabilidade de acoes e configuracao persistida.

## Description
Implementar dois subsistemas de infraestrutura que serao usados por todos os modulos subsequentes:

1. **Sistema de logging** (`src-tauri/src/logging.rs`): logging estruturado para arquivo com rotacao, usando `tracing` + `tracing-appender`. Formato: `[timestamp] [modulo] [acao] [resultado] [detalhes]`. Arquivo em `%APPDATA%\WindowsOptimizer\logs\`.

2. **Config store** (`src-tauri/src/config/app_config.rs`): persistencia de preferencias do app em JSON via `serde_json`. Thresholds de temperatura, paths de scan, modo portable, e outras configuracoes. Exposto via Tauri commands para o frontend.

## Technical Notes

### logging.rs
```rust
use tracing_appender::rolling::{RollingFileAppender, Rotation};
use tracing_subscriber::{fmt, prelude::*, EnvFilter};

pub fn init_logging(data_dir: &std::path::Path) -> Result<(), Box<dyn std::error::Error>> {
    let log_dir = data_dir.join("logs");
    std::fs::create_dir_all(&log_dir)?;

    let file_appender = RollingFileAppender::new(
        Rotation::DAILY,
        log_dir,
        "windows-optimizer",
    );

    tracing_subscriber::registry()
        .with(fmt::layer().with_writer(file_appender))
        .with(EnvFilter::new("info"))
        .init();

    Ok(())
}
```

### AppConfig struct (config/app_config.rs)
```rust
#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub struct AppConfig {
    pub thresholds: TemperatureThresholds,
    pub scan_paths: ScanPaths,
    pub portable_mode: bool,
    pub log_level: String,
}

#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub struct TemperatureThresholds {
    pub cpu_celsius: u32,    // default: 85
    pub gpu_celsius: u32,    // default: 90
    pub disk_celsius: u32,   // default: 60
    pub motherboard_celsius: u32, // default: 80
}

#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub struct ScanPaths {
    pub projects_dirs: Vec<String>, // para Developer Mode: default %USERPROFILE%\Projects
    pub inactive_threshold_days: u32, // default: 90
    pub large_file_threshold_mb: u32, // default: 100
}
```

### Modo portable (NFR-011)
Detectar portable: se o executavel esta rodando de um diretorio que nao e `%PROGRAMFILES%`, usar `{exe_dir}/data/` para logs e config. Funcao `resolve_data_dir()` no `config/app_config.rs`:
```rust
pub fn resolve_data_dir() -> std::path::PathBuf {
    // Se portable: exe_dir/data/
    // Se instalado: %APPDATA%/WindowsOptimizer/
}
```

### Integracao com main.rs
O `init_logging()` e `AppConfig::load()` devem ser chamados cedo no `main.rs`, antes do `tauri::Builder`. O `AppConfig` deve ser registrado como estado Tauri via `.manage(AppState { config: Arc::new(Mutex::new(config)), ... })`.

### Rotacao de logs
`tracing-appender` com `Rotation::DAILY` cria um arquivo por dia. Para o limite de 50 MB e retencao de 30 dias, implementar limpeza no startup: deletar arquivos com mais de 30 dias ou que excedam 50 MB ao iniciar o app.
