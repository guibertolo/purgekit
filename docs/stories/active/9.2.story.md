# Story 9.2: Scan de node_modules Orfaos

## Metadata
- **Epic:** 9 - Developer Mode
- **Status:** Draft
- **Priority:** P0

- **Depends On:** 9.1
- **FR References:** FR-054

## User Story
Como desenvolvedor, eu quero que o app escaneie meus diretorios de projetos em busca de pastas `node_modules` orfas ou inativas, para que eu possa liberar espaco de projetos que nao uso mais.

## Description
Implementar scan recursivo de `node_modules` em diretorios configurados pelo usuario (default: `%USERPROFILE%\Projects`, `%USERPROFILE%\Documents`). Para cada `node_modules` encontrado, classificar como: Ativo (projeto recente), Inativo (sem modificacao > threshold), ou Orfao (sem `package.json` no diretorio pai).

Espaco recuperavel estimado: 30-150 GB em maquinas de dev tipicas (fonte: PRD secao 9).

## Acceptance Criteria
- [ ] AC-1: Tauri command `scan_node_modules` aceita `{ scan_dirs: Vec<String>, inactive_threshold_days: u32 }` e retorna `Vec<NodeModulesEntry>`
- [ ] AC-2: `NodeModulesEntry` inclui: `path: String`, `size_bytes: u64`, `last_modified: i64`, `has_package_json: bool`, `status: NodeModulesStatus`
- [ ] AC-3: `NodeModulesStatus` enum: `Active` (modificado dentro do threshold), `Inactive` (modificado ha mais de X dias), `Orphan` (sem package.json no pai)
- [ ] AC-4: Threshold configuravel — default 90 dias (FR-054)
- [ ] AC-5: Scan usa `walkdir` recursivamente mas para ao encontrar `node_modules` (nao entra dentro de `node_modules` para buscar `node_modules` aninhados — apenas top-level)
- [ ] AC-6: Tamanho de cada `node_modules` e calculado via `calc_dir_size` da Story 2.1
- [ ] AC-7: Scan emite eventos de progresso `"dev-scan-progress"` com `{ scanned_dirs: u32, found_count: u32 }` durante execucao
- [ ] AC-8: Scan pode ser cancelado (Tauri command `cancel_dev_scan`)
- [ ] AC-9: Resultado inclui `total_size_bytes` somando apenas entradas selecionadas para limpeza (Inactive + Orphan por default)

## Technical Notes

### NodeModulesEntry e NodeModulesStatus
```rust
#[derive(Debug, Clone, serde::Serialize)]
pub enum NodeModulesStatus {
    Active,
    Inactive { days_old: u32 },
    Orphan,
}

#[derive(Debug, Clone, serde::Serialize)]
pub struct NodeModulesEntry {
    pub path: String,
    pub parent_path: String,
    pub size_bytes: u64,
    pub last_modified: i64,  // unix timestamp
    pub has_package_json: bool,
    pub status: NodeModulesStatus,
    pub selected: bool,  // default: true para Inactive e Orphan, false para Active
}
```

### Algoritmo de scan
```rust
pub async fn scan_node_modules(
    app: &AppHandle,
    scan_dirs: &[String],
    inactive_threshold_days: u32,
) -> Vec<NodeModulesEntry> {
    use walkdir::WalkDir;
    let threshold_secs = inactive_threshold_days as u64 * 86400;
    let now = std::time::SystemTime::now()
        .duration_since(std::time::UNIX_EPOCH)
        .unwrap().as_secs();

    let mut results = Vec::new();

    for base_dir in scan_dirs {
        let walker = WalkDir::new(base_dir)
            .max_depth(10)  // nao varrer profundo demais
            .into_iter()
            .filter_entry(|e| {
                // Nao entrar dentro de node_modules (evitar varredura de sub-node_modules)
                e.file_type().is_dir() &&
                !(e.depth() > 0 && e.file_name() == "node_modules")
            });

        for entry in walker.flatten() {
            if entry.file_name() == "node_modules" {
                let nm_path = entry.path();
                let parent = nm_path.parent().unwrap();

                let has_pkg_json = parent.join("package.json").exists();
                let (size, _) = calc_dir_size(nm_path);
                let last_mod = get_dir_last_modified(nm_path);

                let status = if !has_pkg_json {
                    NodeModulesStatus::Orphan
                } else if (now - last_mod) > threshold_secs {
                    NodeModulesStatus::Inactive {
                        days_old: ((now - last_mod) / 86400) as u32
                    }
                } else {
                    NodeModulesStatus::Active
                };

                results.push(NodeModulesEntry { ... });

                // Emitir progresso
                app.emit("dev-scan-progress", ...).ok();
            }
        }
    }
    results
}
```

### get_dir_last_modified()
Usar `std::fs::metadata(path)?.modified()` para obter data de ultima modificacao. Para `node_modules`, verificar o diretorio pai (projeto), nao o `node_modules` em si.

### Performance
`calc_dir_size` de uma `node_modules` grande pode ser lento. Executar em `tokio::spawn_blocking` para nao bloquear o async runtime.

### Cancel token
Usar `Arc<AtomicBool>` como cancel token, armazenado em `AppState`. `cancel_dev_scan` seta para `true`, o loop verifica a cada iteracao.
