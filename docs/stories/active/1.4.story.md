## Metadata

## User Story
Como usuario do Windows Optimizer, eu quero que o app solicite elevacao de privilegios apenas quando necessario, para que eu possa usar funcionalidades basicas sem ser admin e tenha controle claro sobre quando admin e requerido.

## Description
Implementar o mecanismo de elevacao UAC descrito na arquitetura (secao 1.3). O app inicia sem privilegios admin. Quando uma operacao requer elevacao, o backend retorna `AppError::ElevationRequired`. O frontend exibe um dialog explicando por que admin e necessario e oferece "Executar como Admin". Ao confirmar, o app re-lanca elevado via `ShellExecuteW` com verbo `runas`, persistindo o estado da sessao antes do re-lancamento.

## Technical Notes

### platform/elevation.rs
```rust
use windows::Win32::Security::{TOKEN_ELEVATION, TokenElevation, OpenProcessToken};
use windows::Win32::System::Threading::GetCurrentProcess;

pub fn is_elevated() -> bool {
    unsafe {
        let mut token = Default::default();
        if OpenProcessToken(GetCurrentProcess(), TOKEN_QUERY, &mut token).is_err() {
            return false;
        }
        let mut elevation = TOKEN_ELEVATION::default();
        let mut size = std::mem::size_of::<TOKEN_ELEVATION>() as u32;
        windows::Win32::Security::GetTokenInformation(
            token,
            TokenElevation,
            Some(&mut elevation as *mut _ as *mut _),
            size,
            &mut size,
        ).is_ok() && elevation.TokenIsElevated != 0
    }
}
```

### useElevation hook (hooks/useElevation.ts)
```typescript
import { invoke } from '@tauri-apps/api/core';

export function useElevation() {
  const [isElevated, setIsElevated] = useState(false);

  useEffect(() => {
    invoke<boolean>('is_elevated').then(setIsElevated);
  }, []);

  const requestElevation = async (currentPage: string) => {
    await invoke('request_elevation', { currentPage });
  };

  return { isElevated, requestElevation };
}
```

### Uso no frontend
Cada pagina que tem operacoes admin usa `useElevation`. Ao tentar executar uma operacao admin sem elevacao, exibe o `ConfirmDialog` de elevacao (componente da Story 1.2) antes de chamar `requestElevation`. O `ConfirmDialog` mostra qual operacao especifica requer admin.

### Arquitetura de elevacao (decisao do arquiteto)
Re-lancamento do app inteiro (nao helper separado) — ver nota `[AUTO-DECISION]` na secao 1.3 da arquitetura. Justificativa: menor complexidade, projeto pessoal.
