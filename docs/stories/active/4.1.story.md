# Story 4.1: Monitor de CPU em Tempo Real

## Metadata
- **Epic:** 4 - System Monitoring Dashboard
- **Status:** Done
- **Priority:** P0
- **Story Points:** 5
- **Depends On:** 1.3
- **FR References:** FR-040, NFR-009

## User Story
Como usuario do Windows Optimizer, eu quero ver o uso de CPU em tempo real com graficos atualizando a cada 1-2 segundos, para que eu monitore a performance do processador durante gaming ou desenvolvimento.

## Description
Implementar o loop de monitoramento de CPU em `modules/system_monitor.rs` usando `sysinfo` crate. O monitor roda em background via `tokio::spawn` e emite eventos Tauri `"system-metrics"` periodicamente. O frontend, via hook `useSystemMetrics`, assina esses eventos e exibe graficos.

Esta story foca apenas em CPU. O evento `"system-metrics"` sera expandido nas stories 4.2-4.4 para incluir RAM, GPU e temperaturas.

## Acceptance Criteria
- [x] AC-1: Background task inicia automaticamente quando o app abre, emitindo evento `"system-metrics"` a cada 2 segundos (NFR-009)
- [x] AC-2: Payload do evento inclui `cpu: CpuMetrics` com: `usage_percent: f32`, `frequency_mhz: u64`, `per_core_usage: Vec<f32>`, `core_count: u32`, `thread_count: u32`
- [x] AC-3: CPU overhead do monitoramento e inferior a 1% (verificar via Task Manager) (NFR-009)
- [x] AC-4: Monitor roda em thread separada, nao bloqueia UI thread
- [x] AC-5: Hook `useSystemMetrics` em `src/hooks/useSystemMetrics.ts` assina eventos `"system-metrics"` e retorna o ultimo valor recebido
- [x] AC-6: `metricsStore.ts` armazena historico dos ultimos 60 segundos de leituras CPU (para grafico temporal)
- [x] AC-7: Grafico de linha de CPU usa Recharts, atualiza sem flickering visivel
- [x] AC-8: Exibe: % de uso total, frequencia atual em GHz, barras por core (para as primeiras 16 cores)

## Technical Notes

### Background monitor loop (arquitetura secao 1.2)
```rust
// services/monitor_service.rs (ou modules/system_monitor.rs)
use sysinfo::{System, CpuExt, SystemExt};

pub async fn start_monitor_loop(app: AppHandle) {
    let mut sys = System::new_all();
    loop {
        sys.refresh_cpu();
        sys.refresh_memory();

        let metrics = build_system_metrics(&sys);
        app.emit("system-metrics", &metrics).ok();

        tokio::time::sleep(Duration::from_secs(2)).await;
    }
}

// Chamado no main.rs apos Tauri setup:
tauri::async_runtime::spawn(start_monitor_loop(app_handle.clone()));
```

### sysinfo API para CPU
```rust
use sysinfo::{CpuExt, SystemExt};

fn build_cpu_metrics(sys: &System) -> CpuMetrics {
    let cpus = sys.cpus();
    let usage_percent = sys.global_cpu_info().cpu_usage();
    let frequency_mhz = cpus.first().map(|c| c.frequency()).unwrap_or(0);
    let per_core_usage: Vec<f32> = cpus.iter().map(|c| c.cpu_usage()).collect();

    CpuMetrics {
        usage_percent,
        frequency_mhz,
        per_core_usage,
        core_count: sys.physical_core_count().unwrap_or(0) as u32,
        thread_count: cpus.len() as u32,
    }
}
```

### useSystemMetrics hook
```typescript
import { listen } from '@tauri-apps/api/event';
import { useMetricsStore } from '@/stores/metricsStore';

export function useSystemMetrics() {
  const addMetrics = useMetricsStore(s => s.addMetrics);

  useEffect(() => {
    const unlisten = listen<SystemMetrics>('system-metrics', (event) => {
      addMetrics(event.payload);
    });
    return () => { unlisten.then(fn => fn()); };
  }, []);

  return useMetricsStore(s => s.latest);
}
```
