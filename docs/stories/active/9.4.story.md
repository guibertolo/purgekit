## User Story
Como desenvolvedor, eu quero limpar caches de pip, cargo, Go, Maven e Gradle, e compactar discos virtuais WSL2, para que eu recupere o maximo de espaco possivel do meu ambiente de desenvolvimento.

## Description
Implementar limpeza de cinco caches adicionais de ferramentas de desenvolvimento (pip, cargo, Go, Maven, Gradle) e a operacao mais complexa: compactacao de disco virtual WSL2 (`.vhdx`). Compactacao WSL2 requer elevacao admin e desligamento do WSL2.

### WSL2 — deteccao de distros
```rust
pub fn detect_wsl2_distributions() -> Result<Vec<Wsl2DistroInfo>, AppError> {
    let output = Command::new("wsl")
        .args(["--list", "--verbose"])
        .output()?;

    // Parsear output para extrair nomes de distribuicoes
    // Formato tipico: "  * Ubuntu    Running    2"
    // Para cada distro, localizar o vhdx em:
    // %LOCALAPPDATA%\Packages\{distro_package_name}\LocalState\ext4.vhdx
}

#[derive(Debug, serde::Serialize)]
pub struct Wsl2DistroInfo {
    pub name: String,
    pub vhdx_path: String,
    pub vhdx_size_bytes: u64,
    pub status: String,  // "Running", "Stopped"
}
```

### Paths do vhdx por distro
Path tipico: `%LOCALAPPDATA%\Packages\CanonicalGroupLimited.Ubuntu....\LocalState\ext4.vhdx`

Como nao ha mapeamento direto de nome para package name, usar `walkdir` para buscar `ext4.vhdx` dentro de `%LOCALAPPDATA%\Packages\` e correlacionar com a lista do `wsl --list`.

### Compactacao WSL2
```rust
pub async fn compact_wsl2_vhdx(
    app: &AppHandle,
    distro_name: &str,
    vhdx_path: &str,
) -> Result<(), AppError> {
    // 1. Verificar elevacao
    if !is_elevated() {
        return Err(AppError::ElevationRequired { operation: "WSL2 vhdx compaction".into() });
    }

    // 2. Desligar WSL2
    Command::new("wsl").arg("--shutdown").output()?;

    // 3. Compactar via PowerShell Optimize-VHD
    let ps_cmd = format!("Optimize-VHD -Path '{}' -Mode Full", vhdx_path);
    Command::new("powershell")
        .args(["-Command", &ps_cmd])
        .output()?;

    Ok(())
}
```

Alternativa se `Optimize-VHD` nao disponivel (requer Hyper-V feature): usar `diskpart` com script:
```
select vdisk file="{vhdx_path}"
compact vdisk
```
