# Story 3.3: Scan e Limpeza de Shader Cache AMD e DirectX

## Metadata
- **Epic:** 3 - GPU Cache Cleaner
- **Status:** Draft
- **Priority:** P0
- **Story Points:** 3
- **Depends On:** 3.1, 3.2
- **FR References:** FR-014, FR-015, FR-016, FR-017

## User Story
Como usuario do Windows Optimizer, eu quero escanear e limpar os shader caches AMD e DirectX, para que eu recupere espaco de shaders compilados independentemente da marca da minha GPU.

## Description
Expandir `modules/gpu_cleaner.rs` com scan e limpeza de shader cache AMD (DxCache, GLCache) e DirectX Shader Cache (`%LOCALAPPDATA%\D3DSCache\`). O DirectX cache funciona para qualquer GPU. Cache AMD so aparece se GPU AMD detectada.

## Acceptance Criteria
- [ ] AC-1: Scan de AMD shader cache retorna tamanho de:
  - `%LOCALAPPDATA%\AMD\DxCache\`
  - `%LOCALAPPDATA%\AMD\GLCache\`
- [ ] AC-2: Scan de DirectX shader cache retorna tamanho de `%LOCALAPPDATA%\D3DSCache\`
- [ ] AC-3: DirectX shader cache aparece sempre (independente de vendor da GPU)
- [ ] AC-4: AMD shader cache aparece apenas se GPU AMD detectada (via Story 3.1)
- [ ] AC-5: Limpeza de nenhum dos dois requer elevacao admin
- [ ] AC-6: `scan_gpu_caches` Tauri command retorna lista com: NVIDIA (se aplicavel), AMD (se aplicavel), DirectX (sempre)
- [ ] AC-7: Warning de primeiro boot mais lento aplicado tambem a AMD e DirectX shader caches (FR-017)

- AMD: `has_amd: bool` derivado da deteccao de GPU (Story 3.1)
