# Story 2.2: Scanner de Browser Cache

## Metadata
- **Epic:** 2 - System Cache Cleaner
- **Status:** Draft
- **Priority:** P0
- **Story Points:** 5
- **Depends On:** 2.1
- **FR References:** FR-006, FR-011

## User Story
Como usuario do Windows Optimizer, eu quero que o app detecte automaticamente quais browsers estao instalados e mostre o tamanho do cache de cada um, para que eu possa limpar o cache dos browsers sem abrir cada um manualmente.

## Description
Expandir o `CacheCleaner` com deteccao e scan de cache de browsers: Chrome, Edge, Firefox e Opera. A deteccao verifica se o browser esta instalado checando a existencia dos paths de cache. Se o browser estiver aberto (processo rodando), exibe aviso mas nao bloqueia o scan. Arquivos em uso sao ignorados durante limpeza.

## Acceptance Criteria
- [ ] AC-1: Browsers detectados automaticamente pela existencia dos paths de cache (Chrome, Edge, Firefox, Opera)
- [ ] AC-2: Para cada browser detectado, `CacheCategory` inclui: nome do browser, path do cache, `size_bytes`, `file_count`, `browser_running: bool`
- [ ] AC-3: Browsers nao instalados (path nao existe) nao aparecem no resultado de scan
- [ ] AC-4: Chrome detectado em `%LOCALAPPDATA%\Google\Chrome\User Data\Default\Cache\`
- [ ] AC-5: Edge detectado em `%LOCALAPPDATA%\Microsoft\Edge\User Data\Default\Cache\`
- [ ] AC-6: Firefox detectado em `%LOCALAPPDATA%\Mozilla\Firefox\Profiles\*\cache2\` (glob para multiplos profiles)
- [ ] AC-7: Opera detectado em `%APPDATA%\Opera Software\Opera Stable\Cache\`
- [ ] AC-8: `CacheCategory.browser_running` e `true` se o processo do browser estiver rodando (Chrome.exe, msedge.exe, firefox.exe, opera.exe)
- [ ] AC-9: O scan de browser cache nao requer elevacao admin (todos os paths sao do usuario)
- [ ] AC-10: Scan de todos os browsers completa em menos de 15 segundos

## Technical Notes

### Paths de cache dos browsers (FR-006)
```rust
const BROWSER_CACHE_PATHS: &[(&str, &str, &str)] = &[
    // (id, display_name, path_template)
    ("chrome",  "Google Chrome", r"%LOCALAPPDATA%\Google\Chrome\User Data\Default\Cache"),
    ("edge",    "Microsoft Edge", r"%LOCALAPPDATA%\Microsoft\Edge\User Data\Default\Cache"),
    ("opera",   "Opera",          r"%APPDATA%\Opera Software\Opera Stable\Cache"),
    // Firefox usa glob: %LOCALAPPDATA%\Mozilla\Firefox\Profiles\*\cache2
];
```

### Firefox — multiplos profiles
Firefox pode ter multiplos profiles em `%LOCALAPPDATA%\Mozilla\Firefox\Profiles\`. Usar `walkdir` ou `std::fs::read_dir` para enumerar subdiretorios e somar os `cache2/` encontrados. Reportar como uma unica categoria "Firefox" com tamanho total.

### Deteccao de browser aberto
```rust
use sysinfo::System;

pub fn is_process_running(process_name: &str) -> bool {
    let sys = System::new_all();
    sys.processes_by_name(process_name).next().is_some()
}

// Nomes de processo por browser:
// chrome.exe, msedge.exe, firefox.exe, opera.exe
```

### Integracao em cache_cleaner.rs
Adicionar metodo `scan_browser_caches()` que retorna `Vec<CacheCategory>` com uma categoria por browser detectado. Integrar com `scan()` geral.

### CacheCategory expandida
Adicionar campo `browser_running: bool` na struct definida na Story 2.1. Default `false` para categorias nao-browser.

### Limpeza (para Story 2.5)
Arquivo em uso durante limpeza de browser: usar `safe_delete_dir_contents` da Story 2.1 que ignora locked files. Se `browser_running: true`, apenas os arquivos nao em uso serao deletados.
