# Story 4.2: Monitor de RAM em Tempo Real

## Metadata
- **Epic:** 4 - System Monitoring Dashboard
- **Status:** Done
- **Priority:** P0

- **Depends On:** 4.1

## User Story
Como usuario do Windows Optimizer, eu quero ver o uso de RAM em tempo real com barra de uso e valores numericos, para que eu monitore o consumo de memoria durante gaming ou desenvolvimento.

## Description
Expandir o monitor loop da Story 4.1 com metricas de RAM. Adicionar `RamMetrics` ao `SystemMetrics` emitido via evento `"system-metrics"`. Exibir no Dashboard: barra de uso, GB usada/total, RAM cached e commit charge.

## Acceptance Criteria
- [ ] AC-1: Evento `"system-metrics"` inclui campo `ram: RamMetrics`
- [ ] AC-2: `RamMetrics` contem: `used_bytes: u64`, `total_bytes: u64`, `available_bytes: u64`, `cached_bytes: u64`, `usage_percent: f32`
- [ ] AC-3: Valores batem com os exibidos no Task Manager do Windows (margem de 1-2%)
- [ ] AC-4: Dashboard exibe barra visual de RAM: preenche proporcionalmente ao uso, cor verde < 70%, amarelo 70-90%, vermelho > 90%
- [ ] AC-5: Dashboard exibe: "X.X GB / Y.Y GB (Z%)" em formato JetBrains Mono
- [ ] AC-6: Historico de RAM armazenado no `metricsStore` para grafico temporal (mesmo padrao do CPU — 30 amostras)

## Technical Notes

### RamMetrics struct
```rust
#[derive(Debug, Clone, serde::Serialize)]
pub struct RamMetrics {
    pub used_bytes: u64,
    pub total_bytes: u64,
    pub available_bytes: u64,
    pub cached_bytes: u64,
    pub usage_percent: f32,
}
```

### sysinfo API para RAM
```rust
use sysinfo::SystemExt;

fn build_ram_metrics(sys: &System) -> RamMetrics {
    let total = sys.total_memory();
    let used = sys.used_memory();
    let available = sys.available_memory();

    RamMetrics {
        used_bytes: used,
        total_bytes: total,
        available_bytes: available,
        cached_bytes: 0, // sysinfo nao expoe cached diretamente; pode ser omitido ou 0
        usage_percent: (used as f32 / total as f32) * 100.0,
    }
}
```

Nota: `sysinfo` nao expoe `cached_bytes` diretamente no Windows. Alternativa: usar `GlobalMemoryStatusEx` via `windows` crate para `ullAvailPageFile` (commit charge). Pode deixar `cached_bytes: 0` inicialmente e melhorar se necessario.
