# Story 10.3: Gaming Mode — GPU Cache, RAM Flush, Game DVR e Prioridade de Processo

## Metadata
- **Epic:** 10 - Gaming Mode
- **Status:** Draft
- **Priority:** P0
- **Story Points:** 8
- **Depends On:** 10.1, 10.2, 3.3
- **FR References:** FR-065, FR-066, FR-067, FR-070

## User Story
Como usuario gamer, eu quero que o Gaming Mode limpe o shader cache, flush de RAM, desative Game DVR e configure a prioridade do processo do jogo, para que eu tenha o maximo de performance possivel durante a sessao de jogo.

## Description
Implementar os quatro recursos restantes do Gaming Mode que complementam a desativacao de servicos (Story 10.2): limpeza opcional de GPU shader cache pre-gaming, flush automatico de working sets de processos idle, desativacao de Game DVR/Xbox Game Bar via Registry, e configuracao de prioridade de processo HIGH para o executavel do jogo selecionado.

Esta story agrupa FR-065, FR-066, FR-067 e FR-070 por serem tecnicamente relacionados e de menor complexidade individual.

## Acceptance Criteria

### FR-065 — GPU Shader Cache pre-gaming
- [ ] AC-1: Gaming Mode inclui opcao toggle "Limpar shader cache antes do jogo" (habilitado por default)
- [ ] AC-2: Se habilitado, executa limpeza de shader caches NVIDIA/AMD/DirectX ao ativar (reutiliza `GpuCleaner` da Story 3.3)
- [ ] AC-3: Aviso incluido no summary: "Primeiros minutos do jogo podem ter maior loading time por recompilacao de shaders"

### FR-066 — RAM Flush
- [ ] AC-4: Gaming Mode executa flush de working sets de processos nao-essenciais ao ativar. Processos nao-essenciais sao definidos como: PID > 10, nao svchost.exe, nao csrss.exe, nao lsass.exe, nao dwm.exe, nao explorer.exe, nao o proprio windows-optimizer.exe
- [ ] AC-5: Usa `EmptyWorkingSet` Win32 API para cada processo nao-essencial (conforme lista de exclusao do AC-4)
- [ ] AC-6: Requer elevacao admin
- [ ] AC-7: `GamingActivateResult` inclui `ram_freed_mb: u32` — calculado como soma dos working sets dos processos antes do flush (estimativa superior, pois nem toda memoria sera realmente liberada ao page file)

### FR-067 — Prioridade de processo
- [ ] AC-8: Tauri command `set_game_process_priority` aceita `{ pid: u32 }` e seta prioridade `HIGH_PRIORITY_CLASS` via `SetPriorityClass`
- [ ] AC-9: Prioridade `REALTIME_PRIORITY_CLASS` nao e oferecida (aviso sobre risco de travamento)
- [ ] AC-10: Ao desativar Gaming Mode, restaura prioridade do processo do jogo para `NORMAL_PRIORITY_CLASS`

### FR-070 — Game DVR desativacao
- [ ] AC-11: Ao ativar Gaming Mode, salva valores originais no `GamingSnapshot.registry_entries` e seta:
  - `HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\GameDVR\AppCaptureEnabled = 0` (DWORD)
  - `HKCU\System\GameConfigStore\GameDVR_Enabled = 0` (DWORD)
- [ ] AC-12: Ao desativar, restaura valores originais via Registry snapshot

## Technical Notes

### RAM Flush (FR-066) — platform/process.rs
```rust
use windows::Win32::System::ProcessStatus::EmptyWorkingSet;
use windows::Win32::System::Threading::{OpenProcess, PROCESS_ALL_ACCESS};

pub fn flush_working_sets_idle() -> Result<u64, AppError> {
    let sys = sysinfo::System::new_all();
    let critical_pids: HashSet<u32> = get_critical_pids(); // PID 0, 4 (System), etc.

    let mut total_freed = 0u64;

    for (pid, process) in sys.processes() {
        let pid_u32 = pid.as_u32();
        if critical_pids.contains(&pid_u32) { continue; }

        unsafe {
            if let Ok(handle) = OpenProcess(PROCESS_ALL_ACCESS, false, pid_u32) {
                let before = process.memory();
                let _ = EmptyWorkingSet(handle); // ignora erros por processo
                // Nota: nao da para medir "freed" diretamente, usa valor antes como estimativa
                total_freed += before;
            }
        }
    }

    Ok(total_freed)
}
```

### Game DVR via Registry (FR-070) — platform/registry.rs
```rust
use winreg::enums::{HKEY_CURRENT_USER, REG_DWORD};

pub fn disable_game_dvr() -> Result<Vec<RegistrySnapshot>, AppError> {
    let mut snapshots = Vec::new();

    // Salvar e modificar AppCaptureEnabled
    let key = RegKey::predef(HKEY_CURRENT_USER)
        .open_subkey_with_flags(
            r"SOFTWARE\Microsoft\Windows\CurrentVersion\GameDVR",
            winreg::enums::KEY_READ | winreg::enums::KEY_WRITE,
        )?;

    // Salvar valor original
    let original: u32 = key.get_value("AppCaptureEnabled").unwrap_or(1);
    snapshots.push(RegistrySnapshot {
        hive: "HKCU".into(),
        path: r"SOFTWARE\Microsoft\Windows\CurrentVersion\GameDVR".into(),
        value_name: "AppCaptureEnabled".into(),
        original_value: original.to_string(),
        original_type: "DWORD".into(),
    });

    // Setar para 0
    key.set_value("AppCaptureEnabled", &0u32)?;
```

### SetPriorityClass (FR-067)
```rust
use windows::Win32::System::Threading::{SetPriorityClass, HIGH_PRIORITY_CLASS, NORMAL_PRIORITY_CLASS};

pub fn set_process_priority_high(pid: u32) -> Result<(), AppError> {
    unsafe {
        let handle = OpenProcess(PROCESS_SET_INFORMATION, false, pid)?;
        SetPriorityClass(handle, HIGH_PRIORITY_CLASS)?;
        Ok(())
    }
}
```
