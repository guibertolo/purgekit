# Story 2.1: Scanner de Temp Files

## Metadata
- **Epic:** 2 - System Cache Cleaner
- **Status:** Draft
- **Priority:** P0
- **Story Points:** 5
- **Depends On:** 1.3, 1.4
- **FR References:** FR-001, FR-002, FR-011

## User Story
Como usuario do Windows Optimizer, eu quero escanear arquivos temporarios do usuario e do Windows e ver o espaco ocupado por cada categoria, para que eu possa decidir informadamente o que limpar.

## Description
Implementar o modulo `modules/cache_cleaner.rs` com a funcionalidade de scan de temp files: diretorio do usuario (`%TEMP%`) e do Windows (`C:\Windows\Temp`). O scan retorna o tamanho total por categoria, contagem de arquivos, e lista de arquivos em uso que serao ignorados. A limpeza nao acontece nesta story — apenas o scan e a estrutura de dados.

Tambem implementar a plataforma de suporte em `platform/filesystem.rs`: scan recursivo de diretorios com calculo de tamanho, deteccao de arquivos em uso (tentativa de open exclusivo), e delecao segura.

## Acceptance Criteria
- [ ] AC-1: Tauri command `scan_system_cache` retorna `ScanResult` com lista de `CacheCategory` cada uma com: nome, path, `size_bytes`, `file_count`
- [ ] AC-2: Categoria "User Temp" (`%TEMP%`) e incluida no scan sem elevacao admin
- [ ] AC-3: Categoria "Windows Temp" (`C:\Windows\Temp`) e incluida no scan — se sem elevacao, retorna `size_bytes: 0` com flag `requires_elevation: true`
- [ ] AC-4: `ScanResult.total_size_bytes` e a soma de todas as categorias
- [ ] AC-5: O scan completa em menos de 10 segundos para caches tipicos (< 10k arquivos)
- [ ] AC-6: Arquivos em uso (locked) sao detectados e listados em `CacheCategory.locked_files` mas nao causam erro no scan
- [ ] AC-7: Estrutura `platform/filesystem.rs` tem funcoes: `scan_dir(path) -> DirScanResult`, `calc_dir_size(path) -> u64`, `safe_delete_file(path) -> Result<(), AppError>`
- [ ] AC-8: Todos os paths resolvem variaveis de ambiente (`%TEMP%`, `%WINDIR%`) corretamente

### Resolucao de variaveis de ambiente
Usar `std::env::var("TEMP")` para `%TEMP%`, `std::env::var("WINDIR")` para `%WINDIR%`, etc. Alternativa: usar crate `directories` para paths padrao.

### Crates usados
- `walkdir` para scan recursivo (ja no Cargo.toml da Story 1.1)
- `std::fs` para operacoes de arquivo
- `windows` crate para arquivos locked (Win32 file sharing)

### Command em commands/cache.rs
```rust
#[tauri::command]
pub async fn scan_system_cache(
    state: State<'_, AppState>,
    categories: Option<Vec<String>>, // None = todas
) -> Result<ScanResult, AppError> {
    let cleaner = CacheCleaner::new(&state.config);
    cleaner.scan(categories).await
}
```
